# Исправление ошибок с недействительными FCM токенами

## ✅ Хорошие новости

Ошибка изменилась с `Permission denied` на `Requested entity was not found` / `not a valid FCM registration token`. Это означает:
- ✅ Firebase проект настроен правильно (`ssboss-940a1`)
- ✅ Service account имеет права доступа
- ✅ Push-уведомления пытаются отправляться
- ✅ Код автоматически удаляет недействительные токены из базы данных

## Проблема

FCM токен в базе данных недействителен или устарел. Ошибки могут быть:
- `Requested entity was not found` - токен не найден в Firebase (удален или недействителен)
- `not a valid FCM registration token` - токен имеет неправильный формат
- `InvalidRegistration` - токен недействителен
- `MismatchSenderId` - токен был создан для другого проекта Firebase

Это может произойти, если:
1. Приложение было переустановлено
2. Токен был сгенерирован для другого проекта Firebase
3. Токен устарел (Firebase периодически обновляет токены)
4. Приложение не зарегистрировало новый токен после обновления
5. Токен был удален из Firebase (например, при переустановке приложения)

## Решение

### Вариант 1: Обновить токен через мобильное приложение (РЕКОМЕНДУЕТСЯ)

1. **Откройте мобильное приложение**
2. **Войдите в аккаунт** (user_id: 3)
3. **Приложение автоматически зарегистрирует новый FCM токен** при запуске
4. Проверьте логи сервера - должен появиться:
   ```
   [INFO] FCM token updated for user 3
   ```

### Вариант 2: Проверить токен в базе данных

```bash
# Подключиться к MySQL
mysql -u root -p

# Выбрать базу данных
USE ssboss_db;

# Проверить токен пользователя
SELECT id, email, fcm_token FROM users WHERE id = 3;

# Если токен есть, но недействителен, можно удалить его
UPDATE users SET fcm_token = NULL WHERE id = 3;
```

После удаления токена, при следующем запуске приложения оно автоматически зарегистрирует новый токен.

### Вариант 3: Принудительно перерегистрировать токен

Если приложение не регистрирует токен автоматически:

1. **В мобильном приложении:**
   - Выйдите из аккаунта
   - Закройте приложение полностью
   - Откройте приложение снова
   - Войдите в аккаунт

2. **Проверьте логи приложения** - должен появиться новый FCM токен

3. **Проверьте логи сервера:**
   ```bash
   tail -f storage/logs/laravel.log | grep "FCM token"
   ```

## Автоматическая обработка недействительных токенов

Код теперь автоматически удаляет недействительные токены из базы данных при ошибке отправки. Это означает:
- При следующей попытке отправки недействительный токен будет удален
- Пользователю нужно будет перерегистрировать токен через приложение

## Проверка работы

После обновления токена:

1. **Проверьте токен в базе:**
   ```sql
   SELECT id, email, fcm_token FROM users WHERE id = 3;
   ```

2. **Попробуйте отправить тестовое уведомление:**
   - Измените статус заказа в админ-панели
   - Или используйте API endpoint `/api/v1/admin/push-test`

3. **Проверьте логи:**
   ```bash
   tail -n 20 storage/logs/laravel.log | grep -i "push\|fcm"
   ```

   Должны появиться записи:
   ```
   ✅ PushNotificationService: Уведомление отправлено
   ```

## Частые причины недействительных токенов

1. **Приложение было переустановлено** - старый токен больше не работает
2. **Токен был создан для другого проекта Firebase** - несовместим с текущим проектом
3. **Приложение не обновило токен** - нужно перезапустить приложение
4. **Токен устарел** - Firebase периодически обновляет токены

## Предотвращение проблемы

Убедитесь, что мобильное приложение:
1. ✅ Регистрирует FCM токен при каждом запуске
2. ✅ Обновляет токен при изменении (Firebase может обновлять токены)
3. ✅ Отправляет токен на сервер после успешной авторизации

## Диагностика

Если проблема сохраняется:

```bash
# 1. Проверить последние ошибки
tail -n 30 storage/logs/laravel.log | grep -i "fcm\|push\|token"

# 2. Проверить токен в базе
mysql -u root -p -e "USE ssboss_db; SELECT id, email, LEFT(fcm_token, 30) as token_preview FROM users WHERE id = 3;"

# 3. Проверить, регистрируется ли новый токен
tail -f storage/logs/laravel.log | grep "FCM token updated"
```
