# Инструкция по тестированию Push-уведомлений

## Подготовка

### 1. Проверка настроек сервера

Убедитесь, что в `.env` файле указаны (или используйте значения по умолчанию):
```env
FIREBASE_CREDENTIALS=service-account.json
FIREBASE_PROJECT_ID=ssboss-940a1
```

### 2. Проверка файла service-account.json

Файл должен находиться в корне проекта `server_code/service-account.json`

### 3. Проверка логов

Логи будут записываться в `storage/logs/laravel.log`. Убедитесь, что файл доступен для записи.

## Тестирование

### Тест 1: Регистрация FCM токена

**Цель:** Проверить, что мобильное приложение может сохранить FCM токен на сервере.

#### Шаг 1: Получите FCM токен в мобильном приложении

1. Запустите мобильное приложение
2. Войдите в аккаунт (или используйте гостевой режим)
3. Проверьте логи приложения - должен появиться FCM токен:
   ```
   [NOTIFICATIONS] 🔑 FCM токен получен: dK8x9Yz2...
   ```

#### Шаг 2: Отправьте токен на сервер

**Endpoint:** `POST /api/v1/user/fcm-token`

**Headers:**
```
Authorization: Bearer {user_token}
Content-Type: application/json
```

**Body:**
```json
{
  "fcm_token": "dK8x9Yz2...",
  "device_type": "android"
}
```

**Ожидаемый ответ:**
```json
{
  "success": true,
  "data": {
    "message": "FCM token registered",
    "user_id": 123
  }
}
```

**Проверка в базе данных:**
```sql
SELECT id, email, fcm_token FROM users WHERE id = 123;
```
Должен быть заполнен `fcm_token`.

**Проверка логов:**
```bash
tail -f storage/logs/laravel.log | grep "FCM token"
```
Должна быть запись: `FCM token updated for user 123`

### Тест 2: Отправка push-уведомления при изменении статуса заказа

**Цель:** Проверить, что при изменении статуса заказа отправляется push-уведомление.

#### Шаг 1: Создайте тестовый заказ

1. Создайте заказ через мобильное приложение или админ-панель
2. Запомните `order_id` и `user_id` заказа

#### Шаг 2: Убедитесь, что у пользователя есть FCM токен

```sql
SELECT id, email, fcm_token FROM users WHERE id = {user_id};
```

#### Шаг 3: Измените статус заказа через админ-панель

1. Зайдите в админ-панель
2. Найдите заказ
3. Измените статус (например, с "Ожидает обработки" на "Подтверждён")

**Или через API:**

**Endpoint:** `POST /api/v1/admin/order/update-status`

**Headers:**
```
Authorization: Bearer {admin_token}
Content-Type: application/json
```

**Body:**
```json
{
  "id": 123,
  "status": 2
}
```

#### Шаг 4: Проверьте результат

**В мобильном приложении:**
- Должно прийти push-уведомление с текстом: "Ваш заказ #20251911rfZW3 теперь: Подтверждён"
- При нажатии на уведомление должен открыться экран деталей заказа

**В логах сервера:**
```bash
tail -f storage/logs/laravel.log | grep "PushNotificationService"
```
Должна быть запись:
```
PushNotificationService: Уведомление отправлено
```

**Проверка ошибок:**
```bash
tail -f storage/logs/laravel.log | grep "ERROR\|Failed"
```

### Тест 3: Массовая рассылка акций

**Цель:** Проверить отправку push-уведомлений при массовой рассылке акций.

#### Шаг 1: Подготовьте несколько пользователей с FCM токенами

```sql
-- Проверьте, сколько пользователей имеют FCM токены
SELECT COUNT(*) FROM users WHERE fcm_token IS NOT NULL AND fcm_token != '';
```

#### Шаг 2: Создайте формат email для рассылки

1. Зайдите в админ-панель
2. Создайте новый формат email рассылки:
   - **Subject:** "Новая акция! 🎉"
   - **Body:** "Скидка 50% на все товары категории Электроника"

#### Шаг 3: Отправьте массовую рассылку

**Endpoint:** `POST /api/v1/admin/subscriber/send-subscription-email`

**Headers:**
```
Authorization: Bearer {admin_token}
Content-Type: application/json
```

**Body:**
```json
{
  "id": 1
}
```
где `id` - это ID формата email рассылки.

#### Шаг 4: Проверьте результат

**В логах сервера:**
```bash
tail -f storage/logs/laravel.log | grep "Subscription email\|PushNotificationService"
```

Должны быть записи:
```
Subscription email: Push notifications sent
PushNotificationService: Массовая рассылка завершена
```

**В ответе API:**
```json
{
  "success": true,
  "data": {
    "email_sent": true,
    "subscribers_count": 10,
    "push_notifications": {
      "sent": 8,
      "failed": 0,
      "total": 8
    }
  }
}
```

**В мобильных приложениях:**
- Все пользователи с FCM токенами должны получить push-уведомление
- При нажатии должен открыться экран с акцией

## Отладка проблем

### Проблема: FCM токен не сохраняется

**Проверьте:**
1. Правильность авторизации (токен в заголовке)
2. Формат запроса (JSON)
3. Логи сервера на наличие ошибок

**Решение:**
```bash
tail -f storage/logs/laravel.log | grep "FCM token registration"
```

### Проблема: Push-уведомления не приходят

**Проверьте:**
1. FCM токен сохранён в базе данных
2. Firebase credentials правильные
3. Project ID соответствует service-account.json
4. Логи на наличие ошибок

**Решение:**
```bash
# Проверка инициализации Firebase
tail -f storage/logs/laravel.log | grep "PushNotificationService.*инициализации"

# Проверка отправки
tail -f storage/logs/laravel.log | grep "PushNotificationService.*отправлено\|Failed"
```

### Проблема: Ошибка "Invalid token"

**Причины:**
- Токен устарел (пользователь переустановил приложение)
- Токен недействителен

**Решение:**
1. Попросите пользователя перезапустить приложение
2. Токен должен обновиться автоматически
3. Или удалите старый токен и попросите зарегистрировать новый

### Проблема: Ошибка Firebase инициализации

**Проверьте:**
1. Файл `service-account.json` существует и доступен
2. Права доступа к файлу (должен быть читаемым)
3. Project ID правильный

**Решение:**
```bash
# Проверка существования файла
ls -la server_code/service-account.json

# Проверка прав доступа
chmod 644 server_code/service-account.json
```

## Полезные SQL запросы

```sql
-- Проверить пользователей с FCM токенами
SELECT id, email, fcm_token FROM users 
WHERE fcm_token IS NOT NULL AND fcm_token != '';

-- Проверить конкретного пользователя
SELECT id, email, fcm_token FROM users WHERE id = 123;

-- Очистить FCM токен (для тестирования)
UPDATE users SET fcm_token = NULL WHERE id = 123;

-- Подсчитать пользователей с токенами
SELECT COUNT(*) as users_with_tokens FROM users 
WHERE fcm_token IS NOT NULL AND fcm_token != '';
```

## Мониторинг в продакшене

Рекомендуется настроить мониторинг:
1. **Количество успешных отправок** - должно быть > 90%
2. **Количество ошибок** - должно быть минимальным
3. **Время отправки** - не должно превышать 1-2 секунды на уведомление
