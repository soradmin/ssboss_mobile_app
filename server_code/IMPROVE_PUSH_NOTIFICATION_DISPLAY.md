# Улучшение отображения push-уведомлений

## Что было сделано

### 1. Серверная часть (PHP)

**Файл: `app/Services/PushNotificationService.php`**

- Добавлен метод `extractTextFromHtml()` для извлечения чистого текста из HTML
- Обновлен метод `sendPromotionNotification()`:
  - Извлекает текст из HTML для отображения в уведомлении (максимум 200 символов)
  - Сохраняет полный HTML в `data['html_body']` в формате base64 для отображения при нажатии

**Изменения:**
```php
// Извлекаем чистый текст из HTML для уведомления
$plainBody = $this->extractTextFromHtml($body);

// Сохраняем полный HTML в data
$data = [
    'type' => 'promotion',
    'html_body' => base64_encode($body),
];
```

### 2. Клиентская часть (Flutter)

**Файл: `lib/features/notifications/services/notification_service.dart`**

- Добавлен метод `_cleanHtmlFromText()` для очистки HTML из текста уведомления
- Обновлен метод `_showLocalNotification()`:
  - Очищает HTML из body перед отображением в уведомлении
  - Использует `BigTextStyleInformation` для длинных текстов на Android
  - Сохраняет данные в JSON формате для парсинга при нажатии

- Обновлен метод `_handleNotificationData()`:
  - Обрабатывает промо-уведомления с HTML body
  - Декодирует base64 HTML и передает на экран деталей
  - Добавлена обработка ошибок при декодировании

**Новый экран: `lib/features/notifications/screens/notification_details_screen.dart`**

- Отображает заголовок и полное HTML содержимое уведомления
- Использует пакет `flutter_widget_from_html` для красивого отображения HTML
- Добавлена кнопка "Поделиться" для копирования текста в буфер обмена
- Красивый дизайн с карточками и тенями

**Маршрут: `lib/app_router.dart`**

- Добавлен маршрут `/notification-details` для отображения полного сообщения

**Зависимости: `pubspec.yaml`**

- Добавлен пакет `flutter_widget_from_html: ^0.15.2` для отображения HTML

## Как это работает

1. **При отправке рассылки из админки:**
   - Сервер извлекает текст из HTML body
   - Отправляет чистый текст в `notification.body` (для отображения в уведомлении)
   - Сохраняет полный HTML в `data['html_body']` (base64)

2. **При получении уведомления:**
   - В уведомлении отображается чистый текст (без HTML тегов)
   - При нажатии на уведомление открывается экран с полным HTML содержимым

3. **Экран деталей уведомления:**
   - Показывает заголовок и полное HTML содержимое
   - Поддерживает ссылки и форматирование
   - Можно поделиться содержимым

## Установка зависимостей

После обновления кода выполните:

```bash
flutter pub get
```

## Тестирование

1. **Отправьте рассылку из админки:**
   - Зайдите в админку → Subscription → Email formats
   - Выберите формат и нажмите "Send"
   - Убедитесь, что в уведомлении отображается чистый текст (без HTML тегов)

2. **Нажмите на уведомление:**
   - Должен открыться экран с полным HTML содержимым
   - Проверьте, что форматирование отображается правильно
   - Проверьте кнопку "Поделиться"

## Обработка ошибок

- Если HTML body не может быть декодирован, показывается обычный текст
- Если title отсутствует, используется "Уведомление"
- Все ошибки логируются в консоль для отладки
