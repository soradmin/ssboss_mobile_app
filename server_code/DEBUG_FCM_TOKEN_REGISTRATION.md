# Диагностика регистрации FCM токена

## Проблема

FCM токен не регистрируется на сервере. Команда `tail -f storage/logs/laravel.log | grep "FCM token updated"` ничего не показывает.

## Возможные причины

1. **Пользователь не авторизован** - токен отправляется только для авторизованных пользователей
2. **FCM токен не получен** - Firebase не может получить токен
3. **Ошибка при отправке на сервер** - запрос не доходит до сервера
4. **Ошибка авторизации** - токен авторизации недействителен

## Диагностика

### Шаг 1: Проверить логи мобильного приложения

В Android Studio Logcat фильтруйте по:
```
tag:NOTIFICATIONS
```

Ищите записи:
- `🔑 FCM токен получен:` - токен получен от Firebase
- `✅ FCM токен успешно отправлен на сервер` - токен отправлен
- `⚠️ Пользователь не авторизован, токен не отправлен` - пользователь не авторизован
- `❌ Ошибка отправки FCM токена на сервер` - ошибка при отправке

### Шаг 2: Проверить авторизацию пользователя

В мобильном приложении:
1. Откройте приложение
2. **Войдите в аккаунт** (важно!)
3. Проверьте логи - должен появиться FCM токен

### Шаг 3: Проверить все логи сервера (не только FCM)

```bash
tail -f storage/logs/laravel.log
```

Ищите любые записи, связанные с:
- `FCM`
- `fcm-token`
- `user/fcm-token`
- Ошибки авторизации

### Шаг 4: Проверить, что endpoint доступен

```bash
# Проверить последние запросы к API
tail -f storage/logs/laravel.log | grep -i "user/fcm-token\|fcm"
```

### Шаг 5: Проверить токен авторизации

В мобильном приложении проверьте логи:
```
tag:DEBUG | tag:AUTH
```

Ищите:
- `[DEBUG] AuthProvider: Пользователь авторизован`
- `[CONFIG] AppConfig.getMobileBearerToken`

## Решение

### Вариант 1: Пользователь не авторизован

Если в логах приложения видите:
```
⚠️ Пользователь не авторизован, токен не отправлен
```

**Решение:**
1. Войдите в аккаунт в мобильном приложении
2. После входа FCM токен должен автоматически отправиться на сервер
3. Проверьте логи сервера

### Вариант 2: FCM токен не получен

Если в логах приложения НЕТ записи:
```
🔑 FCM токен получен: ...
```

**Возможные причины:**
1. Firebase не инициализирован правильно
2. `google-services.json` не настроен правильно
3. Проблемы с разрешениями на уведомления

**Решение:**
1. Проверьте, что Firebase инициализирован:
   ```
   [NOTIFICATIONS] ✅ Сервис уведомлений инициализирован
   ```
2. Проверьте разрешения на уведомления в настройках устройства
3. Перезапустите приложение

### Вариант 3: Ошибка при отправке на сервер

Если в логах приложения видите:
```
❌ Ошибка отправки FCM токена на сервер: ...
```

**Решение:**
1. Проверьте полный текст ошибки в логах
2. Проверьте, что API endpoint доступен:
   ```bash
   curl -X POST https://ssboss.shop/api/v1/user/fcm-token \
     -H "Authorization: Bearer {token}" \
     -H "Content-Type: application/json" \
     -d '{"fcm_token":"test","device_type":"android"}'
   ```
3. Проверьте логи сервера на ошибки

### Вариант 4: Токен авторизации недействителен

Если запрос доходит до сервера, но возвращает ошибку авторизации:

**Решение:**
1. Выйдите из аккаунта в приложении
2. Войдите снова
3. FCM токен должен отправиться автоматически

## Ручная проверка

### 1. Проверить FCM токен в приложении

В Android Studio Logcat ищите:
```
🔑 FCM токен получен:
```

Скопируйте токен.

### 2. Отправить токен вручную через Postman

**Endpoint:** `POST /api/v1/user/fcm-token`

**Headers:**
```
Authorization: Bearer {ваш_токен_авторизации}
Content-Type: application/json
```

**Body:**
```json
{
  "fcm_token": "скопированный_токен",
  "device_type": "android"
}
```

**Ожидаемый ответ:**
```json
{
  "data": {
    "message": "FCM token registered",
    "user_id": 3
  },
  "status": 200
}
```

### 3. Проверить в базе данных

```sql
SELECT id, email, fcm_token FROM users WHERE id = 3;
```

Если `fcm_token` заполнен, значит токен сохранен.

## Автоматическая отправка токена

FCM токен должен отправляться автоматически:
1. ✅ При инициализации сервиса уведомлений (если пользователь авторизован)
2. ✅ При обновлении токена (если пользователь авторизован)
3. ✅ После успешной авторизации (нужно добавить)

## ✅ Улучшение: Отправка токена после авторизации

**ИСПРАВЛЕНО:** Теперь FCM токен автоматически отправляется на сервер после успешной авторизации.

Если пользователь авторизуется после инициализации сервиса, токен теперь отправляется автоматически.

## Проверка работы

После исправления:

1. **Войдите в аккаунт** в мобильном приложении
2. **Проверьте логи приложения:**
   ```
   🔑 FCM токен получен: ...
   ✅ FCM токен успешно отправлен на сервер
   ```
3. **Проверьте логи сервера:**
   ```bash
   tail -f storage/logs/laravel.log | grep "FCM token updated"
   ```
   Должна появиться запись:
   ```
   [INFO] FCM token updated for user 3
   ```

## Команды для диагностики

```bash
# 1. Проверить все логи, связанные с FCM
tail -n 100 storage/logs/laravel.log | grep -i "fcm\|token"

# 2. Проверить последние запросы к API
tail -n 50 storage/logs/laravel.log | grep "user/fcm-token"

# 3. Проверить ошибки авторизации
tail -n 50 storage/logs/laravel.log | grep -i "unauthorized\|auth\|401"

# 4. Проверить токен в базе данных
mysql -u root -p -e "USE ssboss_db; SELECT id, email, LEFT(fcm_token, 30) as token_preview FROM users WHERE id = 3;"
```
